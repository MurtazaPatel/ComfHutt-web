## Fix 1: ChatInput — Routes to nonexistent `/chat`

### Problem
`src/components/ChatInput.tsx:72` routes to `/chat?q=${query}`, but no `/chat` route exists. The ChatInput is used in `FooterCTA` (landing page) where it should lead users into the scoring flow.

### Fix
- Change `router.push(\`/chat?q=${query}\`)` → `router.push(\`/dashboard?q=${query}\`)`
- In `src/app/dashboard/page.tsx`, read `searchParams` on mount and pre-fill the search input when a `?q=` param is present
- Dashboard already has an auth guard via `useAuth()`, so unauthenticated users get redirected to signin, creating: enter address → signin → dashboard (with address pre-filled)

### Files to change
- `apps/crux/src/components/ChatInput.tsx` — line 72
- `apps/crux/src/app/dashboard/page.tsx` — read `useSearchParams()` for `?q=` and pre-fill `searchInput`

## Fix 6: Unify auth system — Clerk + localStorage mismatch

### Problem
Two auth guards used inconsistently:

| Page | Auth Guard |
|------|-----------|
| `dashboard/page.tsx` | `useAuth()` (Clerk) ✅ |
| `lens/[scoreId]/page.tsx` | `useAuth()` (Clerk) ✅ |
| `score/[scoreId]/page.tsx` | `localStorage.getItem('crux_user')` ❌ |
| `watch/page.tsx` | `localStorage.getItem('crux_user')` ❌ |
| `cast/page.tsx` | `localStorage.getItem('crux_user')` ❌ |
| `yield/page.tsx` | `localStorage.getItem('crux_user')` ❌ |
| `settings/page.tsx` | `localStorage.getItem('crux_user')` ❌ |
| `onboarding/page.tsx` | none ❌ |

If Clerk signs out but `crux_user` stays in localStorage → user sees protected pages but API calls fail. If localStorage is cleared but Clerk valid → user is redirected to signin unnecessarily.

### Fix

1. **Create a shared hook** `src/hooks/useRequireAuth.ts`:

```ts
import { useEffect, useState } from 'react';
import { useAuth } from '@clerk/nextjs';
import { useRouter } from 'next/navigation';

export function useRequireAuth(redirectTo = '/signin') {
  const { isLoaded, isSignedIn } = useAuth();
  const router = useRouter();
  const [ready, setReady] = useState(false);

  useEffect(() => {
    if (isLoaded) {
      if (!isSignedIn) {
        router.push(redirectTo);
      } else {
        setReady(true);
      }
    }
  }, [isLoaded, isSignedIn, redirectTo, router]);

  return { ready, isSignedIn };
}
```

2. **Replace all localStorage guards** with this hook across:
   - `score/[scoreId]/page.tsx`
   - `watch/page.tsx`
   - `cast/page.tsx`
   - `yield/page.tsx`
   - `settings/page.tsx`

3. **Add auth guard to onboarding page** using `useRequireAuth()`

4. **Update `dashboard/page.tsx`** to also use `useRequireAuth()` for consistency (it currently has inline auth logic)

5. **Update the settings page signOut function** — currently it only clears localStorage keys but doesn't call Clerk's `signOut()`. Add `await signOut()` from Clerk.

### Files to change
- **New**: `apps/crux/src/hooks/useRequireAuth.ts`
- `apps/crux/src/app/dashboard/page.tsx` — replace inline auth with hook
- `apps/crux/src/app/lens/[scoreId]/page.tsx` — replace inline auth with hook
- `apps/crux/src/app/score/[scoreId]/page.tsx` — swap localStorage guard for hook
- `apps/crux/src/app/watch/page.tsx` — swap localStorage guard for hook
- `apps/crux/src/app/cast/page.tsx` — swap localStorage guard for hook
- `apps/crux/src/app/yield/page.tsx` — swap localStorage guard for hook
- `apps/crux/src/app/settings/page.tsx` — swap localStorage guard + add Clerk signOut
- `apps/crux/src/app/onboarding/page.tsx` — add auth guard
- `apps/crux/src/components/crux/AppSidebar.tsx` — update `signOut` to also call Clerk
