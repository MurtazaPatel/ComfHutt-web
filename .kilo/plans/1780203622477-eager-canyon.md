### 7. [BUGFIX] ChatInput Placeholder Flicker

**Problem:** When cycling between the 3 placeholder texts ("Enter any address in India...", "Paste a 99acres link...", "Try: 2BHK Satellite, Ahmedabad"), the code sets `displayedPlaceholder` to `""` before the next typing animation starts. This causes a visible flash where you see a partial/empty string.

**Root cause in `ChatInput.tsx` (line 49-53):**
```ts
setPlaceholderIndex((prev) => (prev + 1) % PLACEHOLDERS.length);
setDisplayedPlaceholder("");   // ← immediate clear causes flicker
charIndex = 0;
```

**Fix:** Instead of clearing `displayedPlaceholder`, let the typing sequence naturally overwrite it. Remove `setDisplayedPlaceholder("")` and reset `charIndex` — the `type()` function already handles building the string character by character.

**Files changed:** `apps/crux/src/components/ChatInput.tsx` — only the `setTimeout` callback block inside the useEffect.

## DEFERRED: Inline Styles → Tailwind Conversion

**Why deferred:** Converting 2000+ lines of inline `style={{}}` across 8+ files to Tailwind utility classes while guaranteeing **zero visual changes at any breakpoint** is not feasible in a single pass without visual regression testing at every responsive breakpoint. Inline styles have edge cases (calc(), negative margins, pseudo-elements, vendor prefixes, specificity stacking) that Tailwind utilities sometimes handle slightly differently.

This is best done incrementally, one component per PR, with screenshots at 3 breakpoints (mobile/tablet/desktop) as validation. Not in this round.
