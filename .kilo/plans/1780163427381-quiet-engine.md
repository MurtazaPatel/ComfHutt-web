# Dashboard UI Visual Overhaul Plan

## Objective
Transform the CRUX dashboard into a visually cohesive, professional interface that strictly follows the landing page's established design language — without modifying any landing page files.

## Context
- **Monorepo**: `apps/crux/` is the Next.js app containing both landing and dashboard
- **Design tokens**: Defined in `apps/crux/src/app/globals.css` + `@comfhutt/design-system/tokens.css`
- **Fonts**: Inter (body/sans), DM Serif Display (display/serif), JetBrains Mono (mono)
- **UI library**: shadcn/ui components exist at `apps/crux/src/components/ui/` but are unused in dashboard pages
- **Landing pages**: `apps/crux/src/app/page.tsx` + `components/landing/*` — **DO NOT MODIFY**

### Dashboard Critical Issues Found

#### 1. Typography Chaos
- **Headlines too small**: Dashboard welcome uses `clamp(26px, 3.5vw, 36px)` vs landing hero `clamp(44px, 8vw, 88px)`
- **Arbitrary font sizes**: 9px, 10px, 10.5px, 11px, 11.5px, 12px, 12.5px, 13px, 14px, 15px, 16px, 17px, 18px, 20px, 22px, 26px, 28px, 32px, 36px, 44px, 88px — no scale system
- **Font weights scattered**: 400, 500, 600, 700, 800, extrabold — no hierarchy
- **`crux-display` defined twice** in `globals.css` with conflicting properties (first as generic class, second as `font-family` only)
- **DM Serif Display unused** in dashboard despite being the brand display font
- **Mono font inconsistency**: Sometimes used for labels, sometimes for stats

#### 2. Color Fragmentation
- **Background mismatch**: Dashboard uses `#FAFAF7` (warm off-white), landing uses `#FFFFFF`
- **Primary text dual identity**: `#0A0A1A` vs `#111827` used interchangeably
- **Secondary text quadruplet**: `#6B7280`, `#9CA3AF`, `#D1D5DB`, `#374151` — no semantic mapping
- **Green gradients copy-pasted inline** in ~15 places instead of CSS variables
- **Score band colors** (`#22C55E`, `#F59E0B`, `#EF4444`) hardcoded in 4+ files

#### 3. Spacing & Layout Inconsistencies
- **Max-widths vary per page**: 560px, 600px, 640px, 720px, 960px
- **Page padding varies**: `28px 24px`, `32px 24px`, `40px 24px`, `clamp(24px, 4vw, 48px) clamp(16px, 3vw, 40px)`
- **Header internal padding varies**: 16px vs 20px
- **Card border radii vary**: 16px, 18px, 20px, 22px

#### 4. Massive Code Duplication
- **ScoreRing component**: 4 identical copies in `dashboard/page.tsx`, `lens/page.tsx`, `score/page.tsx`, `dashboard/score/page.tsx`
- **bandColor/bandLabel helpers**: Duplicated in at least 4 files
- **Button gradient styles**: Copy-pasted ~15 times across all dashboard pages
- **Header glassmorphism styles**: Copy-pasted in every single dashboard page

#### 5. shadcn/ui Components Completely Unused
- `Button`, `Card`, `Input`, `Dialog`, `Tabs`, `Badge`, `ScrollArea`, `Sheet`, `Separator`, `Tooltip` all exist in `components/ui/` but dashboard uses raw `<button>` and `<div>` with inline styles

#### 6. Overlayering & Functional Bugs
- **Mobile tab bar overlays content** with no padding compensation on `<main>` element
- **Mobile tab bar padding bug**: `paddingBottom: 'clamp(96px, 14vw, 48px)'` — min (96px) > max (48px), so it resolves incorrectly
- **Mobile tab bar "Lens" href is `/dashboard`** instead of `/lens` — clicking Lens tab goes nowhere new
- **Mobile tab bar primary button overlap**: `marginTop: -18` creates visual z-fighting/overlap with tab bar boundary

#### 7. Visual Hierarchy Weaknesses
- Section titles are `text-base` with `crux-display` — tiny and indistinguishable
- Dashboard welcome lacks visual weight compared to landing hero
- "Your toolkit" and "Recent scores" sections have weak separation
- No consistent page title pattern across dashboard routes

## Implementation Plan

### Phase 1 — Foundation: Fix Globals & Extract Shared Components

#### 1.1 Fix `globals.css` Typography System
- **Remove the duplicate `.crux-display` definition** — consolidate into one proper class
- **Add dashboard typography utilities** that mirror landing page scale:
  - `.dash-hero`: `clamp(32px, 5vw, 48px)`, `font-extrabold`, `letter-spacing: -0.02em`, `line-height: 1.05`, `font-family: var(--font-display)`
  - `.dash-section-title`: `text-[24px] md:text-[32px]`, `font-bold`, `tracking-tight`, `leading-tight`, `font-family: var(--font-display)`
  - `.dash-card-title`: `text-[15px]`, `font-semibold`, `#0A0A1A`
  - `.dash-body`: `text-[14px]`, `#6B7280`, `leading-relaxed`
  - `.dash-label`: `text-[11px]`, `uppercase`, `tracking-[0.16em]`, `font-semibold`, `#9CA3AF`
  - `.dash-eyebrow`: `text-[11px]`, `uppercase`, `tracking-[0.2em]`, `font-semibold`, `#22C55E`
  - `.dash-stat`: `text-[28px]`, `font-extrabold`, `tabular-nums`
- **Standardize font-size scale** to 6 steps: xs(11px) → sm(13px) → base(14px) → md(15px) → lg(18px) → xl(24px) → 2xl(32px)

#### 1.2 Fix `globals.css` Color & Component Utilities
- **Add `.dash-bg`**: `background: #FFFFFF` (match landing)
- **Add `.dash-card`**: `bg-white rounded-2xl border border-[#E5E7EB] shadow-sm p-6` (match landing cards)
- **Add `.dash-card-hover`**: `hover:shadow-md hover:-translate-y-0.5 transition-all duration-200`
- **Add `.dash-btn-primary`**: gradient + `rounded-xl` + shadow + hover brightness
- **Add `.dash-btn-secondary`**: `rounded-xl border-2 border-[#22C55E] text-[#22C55E]`
- **Add `.dash-pill`**: `rounded-full px-3 py-1 text-xs font-medium`
- **Add `.dash-glass-header`**: consolidated glassmorphism header style
- **Fix `.crux-display`**: keep `font-family: var(--font-display)` only, remove the conflicting `font-size`/`line-height` definition

#### 1.3 Create Shared Dashboard Components

**`apps/crux/src/components/crux/ScoreRing.tsx`**
- Single source of truth for animated score ring
- Props: `score`, `size`, `stroke`, `delay`, `className`
- Uses `bandColor` internally

**`apps/crux/src/components/crux/BandBadge.tsx`**
- Displays score band label + dot
- Props: `score`, `size`
- Uses centralized `bandColor`/`bandLabel`

**`apps/crux/src/components/crux/DashboardHeader.tsx`**
- Consistent glassmorphism header for all dashboard pages
- Props: `title`, `icon?`, `badge?`, `backHref?`, `children?` (right-side actions)
- Replaces duplicated header code in 7 files

**`apps/crux/src/components/crux/DashboardShell.tsx`**
- Wraps `AppSidebar` + main content area
- Handles mobile tab bar spacing compensation
- Props: `children`, `mobileTabBar?`
- Sets consistent `max-width`, `padding`, `background`

**`apps/crux/src/components/crux/MobileTabBar.tsx`**
- Extracted from `dashboard/page.tsx`
- **Fix**: href for Lens → `/lens`
- **Fix**: remove `marginTop: -18` overlap
- **Fix**: add proper safe-area padding
- Uses `fixed` positioning with `pb-safe`

**`apps/crux/src/components/crux/DimBar.tsx`**
- Animated dimension progress bar
- Props: `category`, `index`
- Replaces duplicated `DimBar` in `score/page.tsx`, `dashboard/score/page.tsx`, `lens/page.tsx`

#### 1.4 Create Shared Helpers

**`apps/crux/src/lib/score.ts`**
- Export `bandColor(score: number): string`
- Export `bandLabel(score: number): string`
- Export `DIM_ICONS` mapping
- Export `iconFor(name: string)` helper

### Phase 4 — Visual Enhancements

#### 4.1 Apply Landing Card Elevation
- All dashboard cards: `bg-white rounded-2xl border border-[#E5E7EB] shadow-sm`
- Hover state: `hover:shadow-md hover:-translate-y-0.5 transition-all duration-200`
- This directly matches landing page category cards and pricing cards

#### 4.2 Apply Landing Button Styling
- Primary: green gradient, `rounded-xl` or `rounded-full`, white text, `font-semibold text-sm`, green shadow
- Secondary: `rounded-xl`, `border-2 border-[#22C55E]`, `text-[#22C55E]`, `hover:bg-[#22C55E]/5`
- Ghost: `rounded-lg`, `bg-[#F9FAFB]`, `border-[#E5E7EB]`, hover green tint

#### 4.3 Improve Visual Hierarchy
- Page titles: `dash-hero` (DM Serif Display, large, bold)
- Section titles: `dash-section-title` (DM Serif Display or Inter bold)
- Section eyebrows: `dash-eyebrow` (green, uppercase, tracked)
- Body text: `dash-body` (Inter, gray, relaxed)
- Stats: `dash-stat` (mono, tabular, extrabold)

#### 4.4 Add Consistent Section Dividers
- Use `h-px bg-[#F3F4F6]` (landing separator style) instead of mixed border colors
- Add eyebrow + title pattern before major sections

#### 4.5 Animations
- Keep `fadeUp` animation but ensure consistent timing: `0.4s cubic-bezier(0.16,1,0.3,1)`
- Add `prefers-reduced-motion` fallbacks where missing
